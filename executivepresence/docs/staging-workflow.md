# Staging Workflow

## Local

The local HTML mockup is for fast iteration and review. It lets us revise messaging, page order, visual hierarchy, and section structure without touching the live Squarespace site.

## Squarespace

For implementation, create duplicate or hidden Squarespace pages and apply approved changes there first. Do not edit live navigation or replace live pages until the staged page has been reviewed.

## Publishing Checklist

- Confirm desktop layout.
- Confirm mobile layout.
- Check all CTAs.
- Check form behavior.
- Check product/cart links if the page touches ecommerce.
- Confirm SEO title and description.
- Confirm no staging-only notes remain visible.
